# User and administrator guide

## Household owner

Create a household, invite members by e-mail, choose roles, add member profiles, and configure optional dietary preferences. Owners and managers create recipes, plans, storage locations, lists, and budgets. A household always retains an owner.

## Household member

Permissions determine whether a member can view or edit plans, recipes, lists, inventory, budgets, nutrition, and invitations. The shopping view is mobile-oriented and reports conflicts rather than overwriting another shopper.

## Planning workflow

Create a plan for any date range, add or generate meals, lock choices that should not change, and review recommendation reasons. Generate a shopping list to combine ingredients and subtract compatible inventory. Mark a meal prepared only after reviewing proposed inventory deductions.

## Nutrition and budgets

Missing data remains visibly missing and is never silently counted as zero. Values are estimates, not medical advice. Do not derive calorie targets for children; enter professionally determined targets manually if needed.

## Administrator

The Django administration site manages users, global ingredients, units, stores, feature flags, offer providers, uncertain product matches, audit events, support-access records, and job status. Administrators do not automatically receive household membership or private-content access. Create a reasoned, expiring support-access record before support work and end it promptly.
